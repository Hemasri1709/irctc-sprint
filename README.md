# 🚆 IRCTC Sprint – Design Engineering & AI Feature Sprint

## Overview

This repository contains my submission for the Kalvium Design Engineering & AI Sprint.

The objective of this sprint is to study the live IRCTC platform, identify real usability issues, document them in detail, and prepare the foundation for proposing engineering and AI-based improvements.

The project is divided into two parts:

- Part A – Problem Discovery
- Part B – Solution Design

## Part A

Part A contains six documented IRCTC usability problems.

### Given Problems

1. Tatkal Booking Crash
2. Search Filters Not Reliable
3. Seat Selection Reset

### Self-Discovered Problems

4. Passenger Details Form is Long and Confusing
5. Review Journey Page lacks Booking Feedback
6. Payment Method Selection is Cluttered

Each problem contains:

- What is broken
- Affected users
- Frequency
- Current user flow
- Exact failure point
